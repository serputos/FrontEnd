Introducción

HTML, siglas de HyperText Markup Language, hace referencia al lenguaje de marcado para la elaboración de páginas web. Es un estándar que define una estructura básica y un código para la definición de contenido de una página web, como texto, imágenes, etc.

Estructura HTML:


          <!DOCTYPE HTML>
          <html lang="en">
            <head>
              <meta charset="utf-8">
              <title>Título</title>
            </head>
            <body>
            </body>
          </html>

Doctype

Doctype es la primera etiqueta que debemos encontrar en un documento HTML.

El Doctype indica al navegador que lenguaje de marcado debe procesar.

Doctype XHTML:


            <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN"
              "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd>

Doctype HTML5:


            <!DOCTYPE html>

Doctype HTML5

  • El Doctype de HTML5 es mucho más sencillo.
  • No es necesario un namespace.
  • Mejora la comprensión del documento.

    <!DOCTYPE html>
    <html>
      <head>
        <meta charset="utf-8">
        <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">
        <title>My Page Title</title>

        <script src="script.js"></script>
        <link rel="stylesheet" type="text/css" href="style.css">
      </head>

      <body>
        <img alt="Image test" src="path/image.jpg">
      </body>
    </html>

Sintaxis HTML

<html> es la etiqueta que incluye el documento y todos los demás elementos. También se denomina elemento raíz.

Puede usar dos atributos para definirlo:

  • lang: información del idioma
  • dir: dirección del texto

<head>

Head es la etiqueta HTML que contiene la información sobre el documento.

Head nos indica:

  • Título de la página.
  • Metadata, información de los datos que contiene.
  • Scripts y estilos, para las funcionalidades y el diseño.

<meta>

La etiqueta <meta> proporciona información sobre el documento HTML

  • Solo es legible por el navegador o programa.
  • Principalmente se utiliza para especificar:
    • Tipo de contenido.
    • Descripción del contenido.
    • Palabras clave.
    • Autor.
    • Última modificación del documento.
  • Meta ‘Charset’ especifica la codificación de caracteres que se utilizará:
    • UTF-8: Unicode
    • ISO-8859-1: Latin
  • Meta ‘Viewport’ indica cómo debe comportarse el navegador móvil dependiendo del dispositivo y su resolución:
    Propiedad Descripción Posibles
    width: Indica el ancho de la página en el dispositivo. device-width, Npx
    height: Indica el ancho de la página en el dispositivo. device-height, Npx
    initial-scale: Indica cual será el zoom inicial de la página. 1, 0.xx
    maximum-scale / minimum-scale: Zoom máximo o mínimo permitido. 1, 0.xx
    user-scalable: Permitir al usuario hacer zoom. yes, no
  • Metadata de propietario.
    • Esta metadata está definida por aplicaciones externas que interactuarán con nuestra Web.
Ejemplo de <head>: 

        <head>
          <meta charset="utf-8">

          <title>HTML</title>

          <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">
          <meta name="author" content="John Doe">
          <meta name="description" content="Introducción al HTML">

          <!-- External Files -->
          <script src="script.js"></script>
          <link rel="stylesheet" type="text/css" href="style.css">

          <!-- In situ code -->
          <style type="text/css">
            body{ background-color: black; }
          </style>
          <script type="text/javascript">
            console.log("Hello World!");
          </script>
        </head>

<body>

Body es el contenedor de los contenidos. Su misión es contener los elementos que dan forma al documento.


Estructura HTML4: 

        <body>
          <div id="header"></div>
          <div id="nav"></div>
          <div class="article">
            <div class="section"></div>
          </div>
          <div id="sidebar"></div>
          <div id="footer"></div>
        </body>
Estructura HTML5: 

        <body>
          <header></header>
          <nav></nav>
          <article>
            <section></section>
          </article>
          <aside></aside>
          <footer></footer>
        </body>

Elementos Básicos de HTML

Párrafos

En HTML para distribuir el texto en párrafos se utiliza la etiqueta <p>. Esta etiqueta tiene abertura y cierre <p></p> y el contenido pudiendo ser texto, imágenes, enlaces, etc. aparece en un bloque, una caja.

Ejemplo: 

          <p>Un párrafo que contiene texto.</p>

Imágenes

<img> es la etiqueta para insertar una imagen en el documento HTML. Para definir correctamente la imagen que queremos mostrar se deben utilizar los siguientes atributos:

  • src: indica la ubicación de la imagen.
  • alt: se define un texto alternativo, para que aparezca en el documento en el caso de que la ubicación de la imagen sea incorrecta.
  • height: define la altura en que queremos mostrar la imagen. Si no se utiliza este atributo, la imagen aparecerá en su altura original.
  • width: define el ancho en que queremos mostrar la imagen. Si no se utiliza este atributo, la imagen aparecerá en su ancho original.
Código: 

          <img src="https://www.computerhope.com/jargon/h/img.gif" width="300" height="150" alt="Primera Imagen">

Enlaces

Los enlaces se definen en el documento HTML con el elemento anchor <a>, que tiene abertura y cierre <a></a> y su contenido será el texto del enlace. Para definir el punto de destino de un vinculo o ambos debemos usar su atributo href junto con otros atributos que mencionaremos en este documento:

  • href: Éste es el único atributo requerido para definir un hipervínculo. Indica el destino del vínculo, que puede ser una URL o un fragmento de URL que es un nombre precedido por el signo (#) y que indica que el destino se encuentra dentro del mismo documento en que nos encontramos. Las URL no están restringidas a documentos Web (basados en http). Las URL pueden usar cualquier protocolo que el navegador sea capaz de manejar. Por ejemplo, file, ftp y mailto, funcionan en la mayoría de los navegadores.
  • name: Este atributo es requerido cuando definimos un enlace interno (o ancla) en una página. El valor del atributo name es similar al del atributo id y debe ser un identificador alfanumérico único en el documento. En la especificación HTML 4.01, podemos usar tanto id como name en un elemento <a>, siempre que ambos tengan idéntico valor.
  • rel: Para anchor que contengan el atributo href, este atributo especifica la relación que hay entre el objeto destino y aquél donde está el vínculo. El valor es una lista, separada por comas, de relaciones. Los valores y su semántica se registrarán por alguna autoridad que tenga sentido para el autor del documento. La relación por defecto, si no se especifica otra, es void. El atributo rel debe usarse sólo si está presente el atributo href.
  • target: Este atributo especifica el marco destino en un vínculo. La información resultante del vínculo se presentará en el marco seleccionado. Tanto los marcos (frame) como los de tipo (inline frame) deben tener un nombre para poder ser referenciados. Sin embargo, hay algunos nombres especiales. Estos incluyen: "_blank" que indica una nueva ventana, "_parent" que indica el marco padre del marco que contiene el código fuente, "_self" que indica el marco donde está el código fuente y "_top" que indica la ventana completa del navegador. 

          <a href="http://www.mozilla.com" target="_blank">Vínculo externo</a>
